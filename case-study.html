MEAN vs. MERN

Creating the same client using
Angular and React

Context

The following are two fullstack applications which share a backend. Both provide users with information about movies, allow users to "favorite" the movies they like and features a login system along with the ability modify data. The difference is that one was made using Angular and the other using React. It was part of the the full-stack immersion course at CareerFoundry and demonstrates my skill in using React, Angular and all the supporting JavaScript and TypeScript libraries.

    • Conclusion and Final Remarks

    React and Angular, while capable of achieving the same goal, take different approaches.

    React allowed for more freeform development, enabling me to express my personal coding style. In contrast, Angular imposed a opinionated structure.

    In a collaborative setting, I found Angular to be the preferable choice. Its code structure led to easy comprehension and would aid with collaboration with team members.

    When working solo, React would be my go-to. Its flexibility granted me the freedom to code using my own style allowing for a more personalized development experience. As my ability in React grew, so did my ability to make the code more readable.

    Both React and Angular offered unique advantages. The opportunity to create the same app using both was invaluable in gaining a comprehensive understanding of their strengths and nuances.
